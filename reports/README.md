# AI 工作报告入口

三个编码 AI 的阶段性报告只写在本目录。项目经理读取本目录与 `api/requests.md` 判断进度，不再依赖聊天转述。

## 固定文件

- `backend-api.md`：后端 / API / 履约内核
- `user-frontend.md`：用户端
- `merchant-frontend.md`：商家端

## 回复格式

每次任务完成后，负责 AI 必须更新自己的报告文件，使用以下结构：

1. 本轮目标
2. 修改文件清单
3. 明确未修改范围
4. `api/requests.md` 变更
5. 编号校验
6. 合同变更
7. 现网可联调 / 不可联调
8. 只读检查范围
9. 阻塞点
10. 下一步建议

禁止只在聊天里汇报，禁止跳过编号校验。
